# Спецификация проекта Clash Of Clans Autobot v7

## 1. Назначение проекта

Проект представляет собой desktop-бота для автоматизации фарма в Clash of Clans через эмулятор. Текущая цель системы — стабильный цикл фарма с удаленным управлением через Telegram, фильтрацией баз по ресурсам, восстановлением после типовых ошибок игры/эмулятора и периодическим запуском макроса проходки стен по счетчику атак.

Бот построен вокруг фиксированного экранного состояния, клавиатурных триггеров и шаблонного распознавания экранов.

## 2. Основные задачи системы

- надежно определять домашний экран и экран боя по шаблонам
- запускать поиск и атаку автоматически
- отбрасывать слабые базы по золоту и эликсиру до высадки армии
- высаживать войска, героев, осадную машину и заклинания по фиксированному профилю
- сдаваться и возвращаться домой по заданной логике
- управляться удаленно через Telegram
- останавливаться при заполнении хранилищ
- пытаться восстанавливаться после зависаний, ошибок и черного экрана
- при необходимости запускать отдельный цикл проходки стен через горячую клавишу

## 3. Стек технологий

### 3.1 Язык и runtime

- Python 3

### 3.2 Основные библиотеки

- `pyautogui`
  - скриншоты
  - поиск шаблонов на экране
  - нажатия клавиш
  - управление мышью

### 3.3 Интеграции и окружение

- BlueStacks
- ADB через бинарник BlueStacks
- shell-скрипты для macOS
- batch-скрипты для Windows

## 4. Архитектура

### 4.1 Главные процессы

#### `bot.py`
Основной цикл фарма.

Отвечает за:
- ожидание домашнего экрана
- запуск поиска боя
- ожидание экрана боя
- OCR-проверку ресурсов
- высадку армии
- выход из боя
- запись счетчика атак
- периодическую проверку хранилищ
- периодический запуск проходки стен через макрос
- интеграцию с recovery watchdog

#### `telegram_control_aiogram.py`
Панель управления ботом через Telegram.

Отвечает за:
- постоянную Telegram-панель
- запуск и остановку `bot.py`
- отображение текущего статуса
- смену spell-режимов
- включение и выключение проходки стен
- настройку числа боев до проходки стен
- отправку ручного скриншота
- console mode
- recovery debug
- автоудаление входящих команд и обычных сообщений через 10 секунд

#### `recovery_watchdog.py`
Подсистема восстановления.

Отвечает за:
- детект черного экрана
- поиск критических error-шаблонов
- детект зависшего цикла
- детект отсутствия прогресса
- детект застывшего экрана
- перезапуск игры через ADB
- перезапуск эмулятора при критической ошибке
- Telegram-уведомления о восстановлении

## 5. Текущая функциональная логика

### 5.1 Screen guards

#### Домашний экран

Используются шаблоны:
- `images/here/*.png`

Логика:
- найден любой шаблон -> считаем, что бот дома

#### Экран боя

Используются шаблоны:
- `images/on_war/*.png`

Логика:
- найден любой шаблон -> считаем, что бой открыт и армия готова

#### Возврат домой

Используются шаблоны:
- `images/go_home*.png`

Логика:
- используется перед финальным возвратом домой в surrender sequence

### 5.2 Поиск боя

Текущая последовательность нажатий:
- `space`
- `e`
- `i`

Реализовано в `attack_runtime/navigation.py`.

### 5.3 Фильтр лута

Перед атакой бот читает ресурсы в бою.

#### Текущие условия допуска

Бой начинается только если:
- золото >= `500000`
- эликсир >= `500000`

Иначе бот делает reroll через `Enter`.

#### Текущие координаты регионов

- золото: `(35, 190, 170, 45)`
- эликсир: `(27, 235, 174, 39)`

#### Что делает OCR-модуль

- делает crop региона
- применяет препроцессинг
- заменяет типичные OCR-ошибки:
  - `$ -> 5`
  - `S/s -> 5`
  - `O/o -> 0`
  - `I/l/| -> 1`
- собирает кандидаты чисел
- пишет подробный debug-log в консоль и Telegram console mode

### 5.4 Профиль атаки

Сейчас активен только один боевой профиль: `default`.

#### Порядок высадки

1. герои
2. battle machine
3. войска
4. заклинания
5. прожатие способностей героев по таймерам

#### Герои

- клавиши: `4`, `5`, `6`, `7`
- высадка: `g`

#### Battle machine

- выбор: `3`
- высадка: `g`

#### Войска

- выбор: `1`, `2`
- высадка: `g`

#### Тайминги способностей героев

- `4` -> 1.0s
- `5` -> 7.0s
- `6` -> 3.0s
- `7` -> 2.0s

### 5.5 Режимы заклинаний

#### `stoneDick`
Текущий основной режим.

Поведение:
- 11 заклинаний
- распределение по заранее заданным точкам карты
- небольшой рандомный jitter
- интервал между кастами 1 секунда

#### `crazyWalls`
Более агрессивный режим.

Поведение:
- 11 заклинаний
- каст в 4 квадратные точки около центра карты
- общее время около 2 секунд
- порядок точек задается через `SPELL_CRAZYWALLS_CAST_PATTERN`

### 5.6 Выход из боя

Текущая последовательность:
- ждать 28 секунд
- нажать `e`
- нажать `o`
- нажать `h`

Перед `h` бот проверяет доступность go-home кнопки.

### 5.7 Проходка стен

Текущая логика проходки стен не является распознаванием стен.

Это отдельный макро-цикл по горячей клавише.

#### Текущая схема

- проходка может быть включена или выключена из Telegram
- число боев до проходки может быть изменено из Telegram
- текущее значение в конфиге: 20 боев
- когда счетчик боев доходит до порога:
  1. бот делает `align_wall_screen()`
  2. нажимает `-`
  3. ждет 95 секунд
  4. возвращается к обычному фарму

Система предполагает, что внутри эмулятора на `-` уже висит заранее настроенная проходка/прокачка стен.

### 5.8 Контроль хранилищ

Каждые 20 атак бот читает суммарные значения золота и эликсира в хранилищах.

#### Правило остановки

Если оба значения больше `29000000`:
- отправляется Telegram-сообщение
- основной цикл останавливается

#### Текущие регионы

- золото: `(1145, 104, 316, 47)`
- эликсир: `(1145, 180, 313, 48)`

### 5.9 Telegram-панель

Текущая панель работает через редактирование одного сообщения.

Поддерживаются:
- старт
- стоп
- скриншот
- console mode
- spell modes
- входной профиль
- recovery debug
- переключение проходки стен
- настройка числа боев до проходки
- ручная команда `/wall`

### 5.10 Recovery

#### Что ловится

- черный экран
- error template
- превышение времени цикла
- отсутствие прогресса
- застывший экран

#### Что делается

- перезапуск игры через ADB
- если найдена критическая ошибка -> перезапуск эмулятора и затем игры
- отправка уведомления в Telegram

## 6. Конфигурационная модель

Вся основная конфигурация сейчас сосредоточена в `config.py`.

Основные блоки:
- эмулятор / ADB
- OCR
- screen guards
- тайминги атаки
- spell modes
- wall macro cycle
- Telegram
- recovery

Живые изменения во время работы идут через `runtime_state.py`.

## 7. Что сейчас стабильно

Наиболее стабильные части проекта:

- основной цикл `bot.py`
- home / battle / go-home guards
- Telegram-панель на `aiogram`
- старт/стоп рабочего процесса
- spell mode switching
- default-атака
- surrender-логика
- включение и выключение проходки стен
- смена интервала проходки стен из Telegram
- recovery-каркас
- storage monitor по счетчику атак

## 8. Что работает, но требует доработки

- OCR золота и эликсира в бою
- OCR значений в хранилищах
- устойчивость шаблонного поиска при изменении масштаба/камеры
- recovery thresholds
- wall-cycle как timed macro зависит от точного поведения эмулятора и keymap

## 9. Слабые места текущей системы

- OCR сильно зависит от качества скрина и точности регионов
- `pyautogui` требует активного окна и правильного фокуса
- template matching чувствителен к масштабу и положению камеры
- recovery зависит от доступности ADB и реакции BlueStacks
- проходка стен сейчас не умеет реально распознавать стены

## 10. Что явно не реализовано как готовая стабильная функция

- настоящий AI-модуль распознавания и обхода стен
- многократно параллельный запуск 2-3 инстансов как законченная архитектура
- полноценный ADB-only боевой режим
- несколько полноценных боевых профилей кроме `default`

## 12. Краткий итог

Сейчас проект — это рабочий single-instance farm bot для Clash of Clans с Telegram-управлением, OCR-фильтром ресурсов, recovery-механикой и отдельным макро-циклом для проходки стен. Главный эксплуатационный риск на текущем этапе — визуальное распознавание чисел и любая логика, завязанная на нестабильное экранное состояние.
