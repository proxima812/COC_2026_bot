## Why These Changes Were Made
- The repository had no persistent project memory files.
- The default guidance needed to be adapted to a Python bot project using aiogram, pyautogui, ADB, OCR, and runtime state.
- Reusable project-specific skills are more useful here than generic Python web stack skills.
- The project folder was not its own git repository, so a separate local `.git` was required before pushing to the new GitHub repo.
- A lightweight local release workflow is needed so release-style pushes can be inspected by commit history and rolled back cleanly.
- The secondary village bots are now out of scope and should not remain in active launch paths or project memory.
- Telegram is no longer the preferred operator interface; the project now has a local web panel for start/stop/status/settings.
- Main launch scripts now derive `PROJECT_DIR` from their own location instead of a stale hardcoded path.
- The clickable macOS app should only open the control interface; the bot itself starts from inside that interface.
- The new long-term architecture is now `adb_bot/*`: capture -> perception -> state -> decision -> action -> verification.

## Important Decisions
- Use `MEMORY.md` only for stable, reusable project context.
