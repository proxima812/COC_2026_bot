# Документация по проекту Clash Of Clans Autobot v7

## 1. Общее описание

Это Python-проект бота для Clash of Clans, который работает через эмулятор и управляется через Telegram.

Бот умеет:
- искать базы
- фильтровать их по ресурсам
- атаковать по фиксированной стратегии
- выходить из боя и возвращаться домой
- останавливать фарм при заполнении хранилищ
- восстанавливаться после типовых сбоев
- запускать отдельный цикл проходки стен по горячей клавише

## 4. Запуск

### macOS команды

После установки shell-функции доступны:

- `run bot`
  - запуск эмулятора/игры и Telegram-бота
- `run bot -test`
  - запуск в test UI режиме
- `run tg`
  - только Telegram-бот, без запуска эмулятора
- `run tg -test`
  - только Telegram-бот в test UI режиме
- `run farm`
  - запуск только `bot.py`
- `run stop`
  - остановка рабочего процесса
- `stop`
  - shortcut на stop-скрипт

### Как работает start/stop

Telegram-бот управляет рабочим процессом через `telegram_ui/process_controller.py`.

Что делает controller:
- запускает `bot.py`
- хранит PID в `.bot_process.pid`
- умеет мягко и жестко останавливать процесс
- восстанавливает состояние после перезапуска панели

## 5. Основной цикл бота

Текущий рабочий цикл в `bot.py`:

1. применяет входной профиль
2. инициализирует recovery watchdog
3. сбрасывает session state
4. ждет домашний экран
5. запускает поиск боя
6. ждет battle-ready экран
7. читает золото и эликсир через OCR
8. если ресурсов мало — reroll базы
9. если ресурсов достаточно:
   - опционально делает screenshot базы
   - выравнивает экран боя
   - высаживает армию
   - сдается и возвращается домой
10. увеличивает счетчик атак
11. каждые 20 атак:
   - проверяет хранилища
12. если включена проходка стен и достигнут порог:
   - делает wall align
   - нажимает `-`
   - ждет 95 секунд
13. продолжает основной цикл

## 7. Заклинания

### Режим `stoneDick`

Текущий основной режим.

Поведение:
- 11 заклинаний
- каст по заранее заданным map points
- интервал 1 секунда
- небольшой jitter

### Режим `crazyWalls`

Более быстрый центральный режим.

Поведение:
- 11 заклинаний
- 4 центральные квадратные точки
- весь цикл примерно за 2 секунды

## 8. OCR и распознавание ресурсов

### OCR в бою

Файл:
- `gold_filter.py`

Читает:
- золото
- эликсир

Порог допуска в атаку:
- золото >= 500000
- эликсир >= 500000

Текущее состояние:
- модуль работает
- но все еще чувствителен к качеству скрина, теням, масштабу и шуму OCR
- debug logging включен

Что пишет в лог:
- распознанное золото
- распознанный эликсир
- кандидаты
- raw OCR строки

### OCR хранилищ

Файл:
- `storage_monitor.py`

Логика:
- каждые 20 атак читает золото и эликсир в хранилищах
- если оба значения выше 29,000,000 — отправляет Telegram-уведомление и останавливает бота

Текущее состояние:
- реализовано
- пригодно к использованию
- точность зависит от стабильности экрана и OCR

## 9. Recovery

Файл:
- `recovery_watchdog.py`

Ловит:
- черный экран
- error-шаблоны
- зависший цикл
- отсутствие прогресса
- застывший экран

Действия:
- перезапуск игры через ADB
- перезапуск эмулятора при критическом error template
- уведомление в Telegram

Текущее состояние:
- архитектурно модуль рабочий
- фактическая надежность зависит от поведения BlueStacks и ADB

## 13. Известные ограничения

- бот сейчас завязан на active-window automation через `pyautogui`
- полноценной multi-instance архитектуры пока нет
- проходка стен сейчас не является настоящим распознаванием стен
- OCR — самый слабый узел системы
- recovery — best effort, а не гарантия восстановления
